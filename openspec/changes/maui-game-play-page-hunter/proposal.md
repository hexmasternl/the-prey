## Why

When a game starts, or when a signed-in player taps **Resume** on the main menu for an in-progress game, they must land on the actual gameplay screen â€” today there is no gameplay destination in the MAUI app; the `maui-game-lobby-page` change only hands off to a navigation seam that nothing yet fulfills. The **hunter** needs a purpose-built operations screen: a full-screen tactical map of the playfield, a clear "wait for the server to commit the game" state, a prominent **head-start countdown** that warns them not to move early (or incur a 10-minute penalty), and â€” once the head start ends â€” a live map showing their own heading and every broadcasting prey as a moving dot so they can hunt.

## What Changes

- Add a **hunter game play page** as the gameplay destination for a player whose role is **Hunter**. The lobby's post-start hand-off seam (and the menu's Resume-into-active-game path) route a hunter here; a non-hunter (prey) is routed to the separate prey page (not part of this change) â€” so the entry point makes the **role decision** and this change owns the **hunter branch**.
- **Full-screen map**: host a Mapsui map (same stack as the area editor) filling the screen, drawing the game's **playfield polygon** as a semi-transparent **red** shape from the game's stored area coordinates.
- **Hunter HUD region (separate spec)**: reserve the bottom-of-screen region for the **hunter HUD** (owned by the separate `hunter-hud` capability); this page hosts/embeds that HUD but does not define its internals.
- **Waiting-for-server overlay**: while the game is armed but not yet committed by the backend sweep (game `Status == "Ready"`), show a non-dismissable overlay telling the hunter to wait for the game server to start the game. It clears automatically when the game transitions to `InProgress`.
- **Hunter head-start overlay**: when the game becomes `InProgress`, the hunter's **head-start delay** begins. Show a distinct overlay with a **large countdown timer** counting down to the moment the hunter may move (`HunterMayMoveAt`), a small caption explaining this is the hunter head-start time, and a **red warning** that moving before the allowed time earns a **10-minute penalty** applied after the head start ends. The overlay closes when the countdown reaches zero.
- **Live tactical map (post-head-start)**: once the overlay closes, the full-screen map shows the playfield shape plus:
  - a **green arrow** at the hunter's current GPS position that **rotates to the device compass heading** (points the real-world direction the hunter faces);
  - every other player (**prey**) as a **red dot** at their last broadcast location (only when a location has been broadcast);
  - a prey that has been **caught/tagged** rendered as a **grey dot** instead of red.
- **Live updates**: subscribe to the game's real-time event channel (**Azure Web PubSub**) while the page is visible â€” requesting a connection URL from the server, opening a WebSocket, and joining the game's group â€” updating prey dot positions, prey caught/out state (redâ†’grey), the `Ready`â†’`InProgress` transition, and game-ended, then unsubscribing when it disappears. An initial status load populates the map immediately; the channel keeps it current.
- **Local position + heading seams**: read the device's **own** live position (for the self arrow) and **compass heading** behind interfaces so the view model stays platform-free and unit-testable. (Reporting the position to the server for others to see is the separate `maui-background-location-tracking` capability; this page only *renders* local position + heading.)
- Extend the game client seam with the reads this page needs â€” a rich in-progress **status** read (playfield coordinates, participants with last-known location + state, hunter id, remaining time, head-start moment, endgame flag) and a **Web PubSub game-channel** seam (connection-URL request + WebSocket) â€” each mapping backend status codes / events to typed results.

## Capabilities

### New Capabilities
- `maui-game-hunter-play`: The hunter game play page reached (as the hunter branch of the gameplay hand-off) when a game is started or resumed. Covers the role-based entry into the hunter page; the full-screen playfield map (red semi-transparent polygon) hosting the hunter HUD region; the waiting-for-server overlay while the game is `Ready`; the hunter head-start overlay (large countdown to `HunterMayMoveAt`, head-start caption, red move-early/10-minute-penalty warning) and its auto-close; the live map with the self **green heading arrow**, **red** prey dots (only when broadcast), and **grey** dots for caught/out preys; the live Web PubSub channel updates and phase transitions (`Ready`â†’`InProgress`, game-ended hand-off) while visible; the local position + compass seams; and the authenticated status read + Web PubSub connection/channel client calls with their result mapping.
